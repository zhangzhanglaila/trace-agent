# 01 · 产品需求

## 一句话定位

**让别人在开发自己的 Agent 时，能真正看清 Agent 内部——跑到哪一步、卡在哪、为什么失败。**

现有工具（LangSmith / Langfuse / Phoenix）告诉你「发生了什么」；AgentTrace 要让**别人的 Agent 变透明**，并进一步告诉你「卡在哪、为什么」。

## 目标用户

正在构建自己 Agent 的开发者，尤其是：

- 用 **LangChain / LangGraph** 搭建 Agent（第一优先接入对象）
- 也覆盖：纯 Python 自定义、OpenAI/Anthropic SDK 直调、其他框架（CrewAI/AutoGen/LlamaIndex，后续）

## 核心痛点（来自用户原话）

> 很多 agent 做出来不知道哪里真正出了问题，不知道卡在了哪一步。

拆成三层需求，**按优先级排序**：

| 优先级 | 需求 | 说明 |
|:---:|------|------|
| **P0** | 单次运行透明化 | 只跑一次，就能看到每一步的输入/输出/耗时/状态，并**自动高亮失败或卡住的那一步**。无需第二次运行对比。 |
| P1 | 实时监控运行中 | Agent 正在跑时实时看到走到哪一步、当前卡在哪，像进度条。适合长时运行的 Agent。 |
| P2 | A/B 对比深化 | 已有能力：换了输入就炸时，对比两次运行定位根因。持续打磨。 |

> 注意：现有产品重心在 **A/B 对比（需两次运行）**。P0 的「单次运行透明化」是当前最大能力缺口，是本轮开发重心。

## 接入方式要求

两条通道都要提供：

1. **零改动 / 一行启用**（默认）：`from agenttrace import enable; enable()`，别人 Agent 代码一字不改。
2. **少量装饰器**（进阶）：在关键函数加 `@trace` 换取更精确的步骤边界与命名。

## P0 验收标准（单次运行透明化）

一个别人写的 LangGraph Agent，在**不做 A/B 对比**的前提下：

- [ ] 跑一次后能列出完整**步骤时间线**：每步名称、输入、输出、状态、耗时。
- [ ] 若某步抛异常 → 自动标为 **失败步骤**，并显示错误信息与所在步骤。
- [ ] 若运行未正常结束（卡住/超时）→ 标出**最后执行到的步骤**（卡点）。
- [ ] 若某步耗时异常偏高 → 标为**慢步骤**（相对基线/阈值）。
- [ ] 产出一份结构化「运行健康报告」（JSON 协议，见 `02-技术架构.md`）。
- [ ] 前端有「单次运行」视图，可视化时间线并高亮问题步骤。
- [ ] 全流程零改动可用；加装饰器后步骤信息更精确。

## 非目标（本轮明确不做）

- 不做多租户/账号体系、云端托管。
- 不做除 LangChain 外框架的深度适配（先留扩展点）。
- 不追求生产级性能优化，先保证「看得清、定位准」。
