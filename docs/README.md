# AgentTrace 文档中心

本目录存放 AgentTrace 的**开发标准文件**：需求、技术、设计规范、执行路线。
所有开发决策以这些文档为准；文档与代码不一致时，先更新文档再改代码。

## 文档索引

| 文档 | 作用 | 何时阅读 |
|------|------|----------|
| [01-产品需求.md](./01-产品需求.md) | 产品定位、目标用户、核心需求、验收标准 | 开始任何功能前 |
| [02-技术架构.md](./02-技术架构.md) | 系统分层、核心模块、数据协议、扩展点 | 动代码前 |
| [03-设计规范.md](./03-设计规范.md) | 编码规范、命名、提交、测试、UI 约定 | 写代码 / 提交前 |
| [04-开发路线图.md](./04-开发路线图.md) | 里程碑拆分、每步范围、当前进度 | 规划 / 认领任务时 |

## 配套目录

- `../开发日志/` —— 每个开发日的完成事项与待办，见 [开发日志说明](../开发日志/README.md)
- `../CLAUDE.md` —— 给 AI 协作者的工作指引与标准文件路径

## 维护约定

1. **单一事实来源**：一个决策只在一处记录，其他地方链接过去，不复制。
2. **改动留痕**：重大变更在文档底部「变更记录」追加一行（日期 + 摘要）。
3. **先文档后代码**：新增能力先在 `01`/`04` 落定范围，再动手。
