# 开发日志

每个**有开发活动的日子**建一个 `YYYY-MM-DD.md`，记录当天**完成事项**与**待办事项**。
用途：让进度可追溯、随时能接上上次的工作、避免遗漏。

## 命名

`开发日志/YYYY-MM-DD.md`，例如 `开发日志/2026-07-22.md`。

## 每日模板

```markdown
# YYYY-MM-DD 开发日志

## ✅ 今日完成
- （对应路线图子步骤，如 M1.1）……

## 🚧 进行中
- ……

## 📋 待办（下次优先）
- ……

## 📝 备注 / 决策
- 遇到的问题、做出的决定、需要与用户确认的点
```

## 约定

1. 完成事项尽量关联 `docs/04-开发路线图.md` 的子步骤编号（如 M1.2），便于对照进度。
2. 当天有决策变更时，同步更新 `docs/` 对应文件的「变更记录」。
3. 待办滚动传递：今天没做完的「待办」，明天开条目继续。
